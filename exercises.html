What is Javascript?

JavaScript != Java

JavasScript is a programming language developed in the late 1990s by Netscape. Microsoft said no.

It's similar to most other programming languages, variables, loops, logic, functions. The difference is the browser engine.

Modern browsers let you practice JavaScript inside the browser in real time. How is this different from a language like C++?

JavaScript is still compiled, but it is compiled in the browser by the browser's JavaScript engine.

This means we can test our JavaScript in the browser before saving it to our .js files!

Logic, Part 1: If Statements

So we can create these variables, but that's useless unless we add a little logic into it. Let's start with 'if', 'else' statements.

  1. 'If' statements let you run some code based on whether or not some condition passes. Here's the syntax: 
    if( SOME CONDITION TO CHECK ) { 
        CODE TO RUN IF CONDITION PASSES 
    }

    Here's an example: 

    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    }
  2. What if we only want something to happen if the 'if' condition fails? 'Else' statements! 
    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    } else {
        var itIsOver = 0;
    }
  3. We've secretly just introduced a NEW data type, BOOLEANS! Booleans are a data type that can only ever have 1 of 2 different values, true or false. Booleans are used in 'if' statements and loops. Here's a few examples of Booleans: 
    true
    false

    In addition to those two, comparison expressions can be evaluated as a boolean. Try putting these in your console and see what you get: 

    10 > 5
    10 > 15
    var myNameString = "My name is Josh Soileau";
    myNameString === "My name is Josh Soileau"
    var myNameString = 123456;
    myNameString === "My name is Josh Soileau"
    (insert explanation of === vs == )

  4. All of these examples of booleans can be used inside an if statement as the condition to check!

Arrays

Let's learn about yet another data type in JavaScript: arrays. Arrays are one of the most useful things in JavaScript, they let you organize things in groups.

  1. Arrays are defined by square brackets []. When the compiler sees those square brackets, it knows to expect an array of items inside the brackets. 
    [ ARRAY ITEMS GO IN HERE ]
  2. The items inside the brackets are all separated by commas, to signify the dividers to the compiler. 
    [ arrayItem, anotherArrayItem, yetAnotherArrayItem ]
    In the above case we have an array of size 3.
  3. Create an array of strings. Make it have at least 4 items in it, and make each item be your favorite topping on a pizza. Save that array to a variable. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
  4. Okay, so we now have an array with values in it. How do we get those values later? We can access arrays using INDEXES
  5. What are indexes? They are slots in the array! But the secret is that the first slot of the array is index "0" (like all things comp sci) (insert explanation of starting at 0 and not 1). Think of it this way: 
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ]

       Index 0    Index 1        Index 2         Index 3        Index 4
  6. So the way we access indexes of the array are by using indexes in the array. Here's the syntax: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[ NUMBER OF INDEX I WANT TO ACCESS ]
  7. To get the first index of the array we would use the 0th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[0];
  8. To get the third index of the array we would use the 2nd index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[2];
  9. How do we get the last item in the array? In this case we know it's the 4th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[4];
  10. But what if we didn't know how long our array was? What if somebody inserts a 6th item in the array without us knowing? You can learn how long an array is by calling ".length" on it like this: 
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ].length
    Which is the same thing as saying: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings.length
    Notice how it returns the number "5". But our last index is index #4! So to get the last element in the array, we could do this: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];

    pizzaToppings[ pizzaToppings.length - 1 ];
  11. It's important to note that arrays don't have to hold just 1 type of data. You can have an array of anything! 
    [ "Cheese", 42, 999 ]
    [ "Rocky", true, false, "Bullwinkle" ]
    You can even have an array of arrays! 
    [ [12, 32], ["Cherry", "Vanilla", false] ]

Logic, Part 2: For Loops!

Sometimes you want to run the same bit of code over and over again. There are a couple of ways to do that in javascript, we are going to start with For loops!

  1. For loops are similar to if statements. You have some parenthesis and some curly braces. The code inside the curly braces is the code that runs during the loop. Here's the syntax: 
    for( BEFORE CODE ; WHILE CONDITION ; AFTER CODE ) {
        CODE TO BE RUN DURING LOOP 
    }

    BEFORE CODE: this is set at the beginning of the loop
    It is usually something like var i=0 or something like that

    WHILE CONDITION: the condition to keep running the loop

    AFTER CODE: this runs after every iteration of the loop
  2. Here's a simple example of for loops: 
    var i;
    var concatNumbersVar = "";
    for( i=0; i<6; i++) {
        concatNumbersVar = concatNumbersVar + i;
    }
    At the top, we create two variables, one isn't defined yet (i), the other is an empty string. The loop starts at i = 0, and loops through as long as i < 6, but increasing i by 1 every time the loop finishes. The code inside the loop takes our empty string, and appends a number on the back of it every time the loop happens (remember what happens when you add a string and a number together?)

All About Functions

Sometimes in programming we have to repeat ourselves multiple times. When it comes to that kind of thing, we have a rule in programming. It's called D.R.Y., DON'T REPEAT YOURSELVES. Luckily, in programming, we have a concept called functions that let us perform repetetive behaviors in a simple way.

  1. Let's take a common use case. We want print out to a greeting to the user. If you did that without a function you would do something like this: 
    var userName = "Josh Soileau";
console.log('Hello, ' + userName);
console.log('It's nice to see you today.');
console.log('Looks like you've got a lot on your plate today.')
console.log('Better get to work!');
  2. But if you had different userNames, you'd have to do all 5 lines over and over again for every user. Or, we could just use a function! Here's the function syntax: 
    var myFunctionName = function( PARAMETER TO GIVE THE FUNCTION ) {
  CODE INSIDE THE FUNCTION
};
    So then, whenever you wanted to run "CODE INSIDE THE FUNCTION", you would just call the function like this: 
    myFunctionName( SOME PARAMETER I WANT TO PASS );
    I know that can be confusing, let's see an example. Here's the code we had before, but this time in function form! 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
    IMPORTANT NOTE: Just writing that code out WILL NOT RUN that code. All we did in that code was tell the computer "If anyone ever calls the function 'greetTheUser', do this code right here". To actually run the code, we have to "call" it, like this: 
    greetTheUser( "Josh Soileau" );
  3. So what is the thing inside of our call to greetTheUser()? That is our "parameter". Functions work a bit like machines, you give them SOMETHING, and they do things with it! In this case, we are "giving" our greetTheUser function a string called "Josh Soileau". Let's look at our function again: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
    As you see, our function expects to be passed something (as you can see because there is a variable inside the parenthesis of the function declaration). We are saying that the greetTheUser function expects to be called and passed an argument, and we are going to call that argument 'userName'. The function doesn't know WHAT it is being passed, just that it is being passed SOMETHING. Exercise: Try calling the function with a different data type, like a number, array, or object. 
    greetTheUser( 12.34 );
greetTheUser( "HAL" );
    As you can see, it treats it the same way! It saves whatever it is being passed into this 'userName' variable, and tries to use it as well as it can.
  4. Functions can also return values that we can save for later. See this example: 
    var timesTwo = function(number) {
    return number * 2;
};

var thisShouldEqualFourteen = timesTwo( 7 );
    The above function expects to receive a number, does some math, and gives back a new number. We can save the result in a variable. Super useful!

All About Scope!

Lets take a look at a concept called "scoping".

  1. Here's our previous function: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
  2. Inside this function we make a variable called "userName". What happens if we try and access that variable somewhere else, like this: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};

console.log( userName );
    What do we get there and why? In JavaScript, variables (and functions) are only available in the context they are defined in.
  3. Think of scoping like each bit of code is a small piece of a neighborhood. Each "function" is a fenced in property. Anything on the inside, stays in the inside. In our example above, the "userName" variable is created inside our fenced in yard. But then we try and use that variable OUTSIDE of the fence, and it doesn't work. That is because OUTSIDE of the fence, we don't know what "userName" is, it is undefined!
  4. The same thing will happen here as well: 
    var prepareForUser = function( userName ) {
  var greetingString = "Hello, " + userName;
  var userAge = 54;
  var userFavoriteColor = "Beige";
};

// ALWAYS REMEMBER TO CALL YOUR FUNCTION!
// Otherwise, the code inside it won't run! 
prepareForUser( "Josh Soileau" );

console.log( greetingString );
console.log( "Your age is " + userAge );
console.log( "Your favorite color is " + userFavoriteColor );
  5. So how do we fix that code above?? Remember how our code is kind of like a neighborhood with fences in it? Well, if we create a variable OUTSIDE of ALL of the fences, then that variable is available everywhere, even INSIDE the fences. That is called a variable being "global". Let's look at an example: 
    var wildNeighborhoodDog = "Fido";
var dogIsFarAway = 1;

var callDog = function( dogName ) {

  if( dogIsFarAway ) {
    console.log( "Here, " + dogName + ", here boy!" );
  } else {
    console.log( "Good boy, " + dogName );
  }

};

callDog( wildNeighborhoodDog );
    Because our "wildNeighborhoodDog" and "dogIsFarAway" variables are GLOBAL, we can use them inside our "callDog" function. Let's try and fix our code from before. We can solve our problem by making the variables global, like this: 
    var greetingString;
var userAge;
var userFavoriteColor;

var prepareForUser = function( userName ) {
  greetingString = "Hello, " + userName;
  userAge = 54;
  userFavoriteColor = "Beige";
};

// ALWAYS REMEMBER TO CALL YOUR FUNCTION!
// Otherwise, the code inside it won't run! 
prepareForUser( "Josh Soileau" );

console.log( greetingString );
console.log( "Your age is " + userAge );
console.log( "Your favorite color is " + userFavoriteColor );
    All we had to do is DECLARE our variables globally. We didn't even need to assign them any value. Just the act of declaring them outside the function meant that we could access them outside the function later on.
  6. Now we know that we can make variables GLOBAL and have access to them everywhere. So why don't we do this for everything?? Take a look at this code below, tell me what the output is: 
    var numberOfBalloons = 99;
var balloonsColor = "Red";

var popOneBalloon = function( balloonsColor ) {

  numberOfBalloons = numberOfBalloons - 1;
  var balloonsColor = "Blue";

};

console.log( numberOfBalloons + " " +  balloonsColor + " balloons go by." );
    What is the output of that code and why? (move the console.log around to learn more)
  7. This type of thing is why it's a bad idea to have too many global variables. By saying that YOUR variable is global, you are effectively preventing other coders from using that variable name in different scopes. If you can, try and do without global variables.

Building an Online Bank

Now it's time for today's big assignment. We are going to build a banking system in our browser. For organizational purposes, this would be better kept inside of a ".js" file. So, open up a text editor (like sublime text) and create a new file on the desktop called banking.js

We are going to be creating a series of functions to handle a series of global variables.

These exercises will intentionally be left unanswered for now, and will be filled in during class as we complete them.

  1. First, create global variables for userName, userAddress, accountNumber, accountOpenDate, and accountBalance. For now, these variables will be strings and numbers. We may change that later on to account for multiple account holders if we have time. Declare your variables at the TOP of your file.
  2. First thing's first. Create a function called printAccountInfo that will print the information for this user's account using console.log. It may be helpful, as well, to create a printBalance function that will ONLY print the balance and not the rest of the info.
  3. After we have the our printAccountInfo function, we need functions to manage our balance. First, we will need a setBalance function, which will do exactly that, set the accountBalance variable to some number that is passed as a parameter to the setBalance function.
  4. Next we will need withdraw() and deposit() functions, which will accept a parameter, and use the setBalance function to either add or remove that parameter from the accountBalance function.
  5. Our customers may also want to change their address. Let's create a setAddress function that will accept a string as a parameter, and set the address variable to that string.
  6. Let's say we are the most kick ass bank in the world, and we are giving awesome interest rates on savings accounts. Let's say we give a 3% interest rate, and we give it EVERY SINGLE DAY. Make a new function called giveInterest which will increase the balance by 3%.

Another important note: this can be accomplished a few different ways (like everything in programming). It can be done with for loops, while loops, you can even do it with recursive functions if you want to get elegant.
  8. If time is left over at this point, change our variable types to ARRAYS, and modify all the functions to be able to handle multiple account holders.
