Here's a list of things you need to do to install and set up SASS:

  1. Install Ruby
  2. Install Node

Media Queries

CSS3 comes with a nifty way of only applying CSS if the device it is rendered on meets specific criteria.

For example, we can apply different CSS to devices with a screen resolution less than 800px than one with a resolution higher than 800px.

This act of changing the way a website looks depending on the device & screen resolution is called 

Responsive Design

Here's what a media query looks like: 

body {
	background: blue;
}

@media (max-width: 600px) {
  	body {
		background: red;
	}
}

In this example, the background of the page would be blue unless the window is smaller than 600px, then it would be red.

So the syntax of a media query looks like this: 

@media ( CONDITION HERE ) {
  	CSS HERE
}

Media Query Mixin

Since we're using SASS, we can make media queries a lot simpler using a SASS Mixin.

Mixin Syntax 

@mixin bp($minMax, $breakpoint) {
	
	@media (#{$minMax}: #{$breakpoint}) {
	    @content;
	}

}

There are 2 ways to animate things through CSS 

CSS Transitions
Keyframe Animations

CSS Transitions work by telling the browser "If a CSS value changes, do it slowly so we can see it change"

For example, if an element has this style: 

.element {
	background: black;
}
and I put this CSS property on it: 
.element {
	background: black;
	transition: background 300ms linear;
}
than any change to the "background" property will happen gradually, for 300ms.

So, if I put it all together and have: 
.element {
	background: black;
	transition: background 300ms linear;

	&:hover {
		background: blue;
	}
}
Then whenever I hover over that element, it will fade the background color from black to blue.

The syntax for the transition property looks like this: 

transition: PROPERTY_TO_TRANSITION TIME TIMING_FUNCTION
PROPERTY_TO_TRANSITION is the CSS property which we want to transition between different values.

TIME is the time in which to make the transition, usually in milliseconds

TIMING_FUNCTION is what tells the computer "at what rate do I make the change" between two CSS values. Click here to learn more about timing functions.

Let's talk about how this works.

The "button" now has a "span" inside of it.

We are putting an absolutely positioned "after" psuedoelement on the button (and relative to the button).

That psuedoelement will have a white background, and will be 100% the width and 100% the height of the button.

Our "transition" will occur on the value for "top" on the psueodelement. It will default to 100%. On hover of the button, it will change to 0%. That is the transition of the "slide" that you see in the images, the transition of the psuedoelement going from all the way at the bottom, up to the top of the element.

Be careful of the z-index issue!
You may notice that our psuedo element covers up our text!
To fix this, we need to force the text to a higher z-index than the psuedo element (that is the reason for an extra "span" element in the HTML).

Keyframe Animations Require 2 Pieces

Click Here for more information on Keyframe Animations. 

A "Keyframe" Declaration with a name
An "Animation" CSS property on an element, 
with the name of the "Keyframe" declaration

First, we create a "keyframe" animation.
These work using this syntax: 

@keyframe ANIMATION_NAME {
	0% {
		BEGINNING CSS VALUES HERE
	}
	100% {
		FINAL CSS VALUES HERE
	}
}

There are, of course, different vendor prefixes you'll need to get these to work in the various browsers. Use the link above to learn more about those.

Looking at the code above, we have a starting point (0%) and an ending point (100%).
The CSS values inside each section will transition between the beginning and ending values as long as they are CSS properties that can have a transition (like color or font-size, not "display" or "position").
Let's look at an example 

@keyframe dark-to-light {
	0% {
		background: black;
	}
	100% {
		background: white;
	}
}

In this example, our background would change from black to white. Also, remember that unless specified, animations will continue forever, so this animation will go from black to white, over and over and over again.

Another cool thing about keyframe animations is that we don't just have to have starting and ending values, we can put different css IN THE MIDDLE AS WELL.
Let's look at another example: 

@keyframe merica {
	0% {
		background: red;
	}
	50% {
		background: white;
	}
	100% {
		background: blue;
	}
}

In this example, we inserted a "50%" point in our keyframe animation. Now, our animation will change the background from red to white to blue!
You can use any percentage values here. They don't even have to be in order!

The other thing we need for animations to work is our "animation" property

Having a keyframe declaration alone doesn't do anything. We apply these animations to elements in CSS using the "animation" declaration like this: 

div {
	animation: dark-to-light 1s;
}

Or 

div {
	animation: dark-to-light 1s infinite;
}

The syntax is important here. 

animation: ANIMATION_NAME how_long_it_should_take HOW_MANY_TIMES_IT_SHOULD_RUN

If you don't supply the number of times for the animation to run, it'll assume you meant for the animation to run indefinitely.
So, for our example above, we want to run the animation with the name "dark-to-light", it should take 1s to run once, and it should run indefinitely.

Just like keyframe declarations, the "animation" property requires vendor prefixes to work properly in all the browsers. See the link at the top of the page for more information.
